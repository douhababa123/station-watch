# 项目启动文档设计说明

**日期：** 2026-05-07  
**状态：** 已批准

---

## 目标

为当前项目补充一套清晰的启动说明，覆盖两层信息：

1. README 提供最短路径的本地启动步骤。
2. docs 提供完整启动手册，包含依赖、启动、构建、预览和常见问题。

---

## 方案

### 1. README 放简版

在 README 顶部增加“本地启动（简版）”区块，只保留用户完成本地运行所需的最少步骤：

- 安装 Node.js
- 安装依赖
- 启动开发服务器
- 打开浏览器访问地址
- 跳转到详细文档

这样用户打开仓库首页即可立刻运行项目，不需要先阅读完整说明。

### 2. docs 放详细版

新增 `docs/startup-guide.md`，包含：

- 适用范围
- 环境要求
- 首次安装
- 开发模式启动
- 生产构建与本地预览
- 常见问题

文档内容以当前项目实际脚本为准：`npm run dev`、`npm run build`、`npm run preview`。

---

## 设计取舍

- 不新建额外目录层级，详细文档直接放在 `docs/` 下，便于查找。
- 不重复维护两套完整说明，README 仅做入口，详细说明集中在 docs。
- 不添加与启动无关的部署、业务说明，保持文档聚焦。

---

## 预期结果

- 新用户可在 1 分钟内通过 README 启动项目。
- 需要排查环境或执行构建时，可直接查看 `docs/startup-guide.md`。
- README 与 docs 互相链接，降低维护成本。
